# 设计部竞品字段实验台使用说明

这份说明写给没有计算机专业基础的使用者。你可以把系统理解成一个“图片风格实验台”：上传竞品图，设置希望 AI 分析的字段，系统会生成每张竞品图的结构分析、整个品类的视觉规范，以及最终用于生图的完整提示词。确认内容没问题后，再一键生成整套商品图片。

## 1. 基本流程

1. 打开 `http://localhost:8010/workbench/`。
2. 创建或选择一个项目。
3. 上传竞品图，建议一次上传 3-10 张同品类图片。
4. 配置并保存 `OUTPUT-C` 字段。
5. 配置并保存 `OUTPUT-D` 字段。
6. 在“一键预生成整套方案”里点击“一键生成 OUTPUT-C / OUTPUT-D / 完整提示词”。
7. 检查页面展示的每张竞品图 `OUTPUT-C`、总 `OUTPUT-D` 和完整提示词。
8. 确认没问题后，点击“确认无误，一键生成整套图片”。
9. 在“生成结果”区域查看图片，点击图片可以放大。

## 2. OUTPUT-C、OUTPUT-D 和提示词

`OUTPUT-C` 是单张竞品图分析，用来描述一张图片里的背景、构图、模特姿势、裁切范围、商品占比、卖点表达等。

`OUTPUT-D` 是整个品类视觉规范，用来总结多张竞品图的共同规律，例如主图规范、副图结构、常见姿势、光线风格、禁忌项。

最终提示词是真正发给生图模型的完整说明。它会组合 `OUTPUT-C`、`OUTPUT-D`、图片目标和补充信息。

## 3. 字段类型说明

`text`：短文本，适合颜色、背景、裁切范围等一句话内容。

`textarea`：长文本，适合构图说明、卖点表达、注意事项。

`single_select`：单选，只能从选项里选一个。选项用英文逗号分隔。

`multi_select`：多选，可输出多个标签。选项用英文逗号分隔。

`number`：数字，适合商品占比、数量、评分。

`boolean`：开关，适合是/否判断，例如是否有文字、是否纯白背景。

`image_ref`：图片引用，适合记录参考图编号或图片用途，一行一个。

`list`：列表字段，适合多条规范、风险点、风格标签，一行一个。

## 4. 字段设置建议

字段不要一次设置太多。第一轮建议每个 OUTPUT 保持 8-15 个字段。

字段名要具体。“风格”不如“背景类型”“构图方式”“商品画面占比”“模特姿势”清楚。

能分类的字段尽量用 `single_select` 或 `multi_select`，这样不同模型输出更容易对比。

需要 AI 详细描述的字段用 `textarea`。

需要沉淀为规范的内容用 `list`，例如主图规范、禁忌项、差异化机会。

## 5. 一键预生成整套方案

如果“每行一个要生成的图片目标”留空，系统会默认生成四类图片：

1. 亚马逊主图
2. 面料/细节图
3. 核心卖点图
4. 场景/穿搭图

也可以自己每行写一个目标，例如：

```text
白底主图，真人模特正面展示
面料细节图，突出弹力和纹理
卖点图，突出双层支撑结构
通勤场景图，展示上身效果
```

点击“一键生成 OUTPUT-C / OUTPUT-D / 完整提示词”后，系统会开始调用 AI。这个过程可能需要几十秒到几分钟。页面顶部出现“请求处理中”就说明请求已经发出，请不要重复点击。

## 6. 确认和生图

先检查 `OUTPUT-C` 是否准确描述了每张竞品图。

再检查 `OUTPUT-D` 是否总结出了品类共同规律，而不是照抄某一张竞品图。

最后检查完整提示词是否符合本次要生成的图片目标。

确认没问题后，点击“确认无误，一键生成整套图片”。系统会把本次 `OUTPUT-C` 和 `OUTPUT-D` 标记为已确认，并按每个图片目标生成图片。

## 7. 常见问题

点击按钮后如果页面顶部显示“请求处理中”，说明请求已经发送，请等待。

如果生成失败，结果区域会显示失败原因。可以调整字段、减少竞品图数量，或更换模型后重试。

如果分析结果太散，说明字段可能太开放，可以增加单选、多选字段。

如果生成图不像亚马逊风格，优先检查 `OUTPUT-D`。

如果图片太小看不清，点击生成结果图片即可放大。
